# CS50 - Introduction to Databases with SQL

- **One to one**: Each row in table A connects to one row in table B.
A common example is one user and one profile.

- **One to many**: One row in table A can connect to many rows in table B.
A common example is one author and many books.

- **Many to many**: Many rows in table A can connect to many rows in table B.
This usually needs a third table to store each link.
A common example is students and classes.



## Relational Databases

### Entity Relationship (ER) Diagrams

Relationship flow:
authors(id, name) -> authored(author_id, book_id) -> books(id, isbn, title)

### Suggested Tables

```sql
-- Main book table
CREATE TABLE books (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    isbn TEXT NOT NULL UNIQUE,
    title TEXT NOT NULL,
    publisher_id INTEGER,
    FOREIGN KEY (publisher_id) REFERENCES publishers(id)
);

-- Ratings for books
-- book_id references books(id)
CREATE TABLE ratings (
    book_id INTEGER PRIMARY KEY,
    rating REAL NOT NULL CHECK (rating >= 0 AND rating <= 5),
    FOREIGN KEY (book_id) REFERENCES books(id)
);

-- Authors
CREATE TABLE authors (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    name TEXT NOT NULL
);

-- Publishers
CREATE TABLE publishers (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    name TEXT NOT NULL
);

-- Translators
CREATE TABLE translators (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    name TEXT NOT NULL
);

-- Many-to-many: which author wrote which book
CREATE TABLE authored (
    author_id INTEGER NOT NULL,
    book_id INTEGER NOT NULL,
    PRIMARY KEY (author_id, book_id),
    FOREIGN KEY (author_id) REFERENCES authors(id),
    FOREIGN KEY (book_id) REFERENCES books(id)
);

-- Many-to-many: which translator translated which book
CREATE TABLE translated (
    translator_id INTEGER NOT NULL,
    book_id INTEGER NOT NULL,
    PRIMARY KEY (translator_id, book_id),
    FOREIGN KEY (translator_id) REFERENCES translators(id),
    FOREIGN KEY (book_id) REFERENCES books(id)
);
```
