## Jottings

A **table** stores some set of information and every **row** in the table stores one item in that set where every **column** has some piece of information about that item.

**Why do we need databases?**
+ Scale: we need to store and manage large amounts of data
+ Frequency: we need to store and manage data that is updated frequently
+ Speed: we need to store and manage data that is accessed frequently

**What is a Database?**
A collection of data, organized for creating, reading, updating, and deleting data.

**Database Management System (DBMS)**
A software that allows you to create, read, update, and delete data in a database. In other words, it is the software via which you can interact with a database.

There are varieties of DBMSs, but the most popular ones are:
+ MySQL
+ Oracle
+ PostgreSQL
+ SQLite
...

Other DBMSs are:
+ MongoDB
+ ...

If we are to decide which to use, we may need to consider the following factors:

+ Proprietary vs Open Source: Proprietary DBMSs are closed-source and closed-source DBMSs are open-source (implementing the SQL standard)
+ Heavyweight vs Lightweight: Heavyweight DBMSs (PostgreSQL, MySQL) are more complex and require more resources to run, while lightweight DBMSs (SQLite) are simpler and require less resources to run.

**SQL (Structured: keywords Query Language: Ask questions)**
A language for interacting with databases. It is used to create, read, update, and delete data in a database.
In other words, it is a language via which you can create, read, update, and delete data in a database.

**Querying**
A process of asking questions about the data in a database.
